# Консольный ИИ-ассистент (“GigaVibeMiptCode”)

Консольный чат-бот на Python для общения с большими языковыми моделями (LLM). Программа поддерживает работу с любым OpenAI-совместимым API (например, локальная Ollama или оригинальный API от OpenAI). 

## Основной функционал
* **Умное управление контекстом:** Бот автоматически следит за размером истории сообщений. При превышении лимитов (по количеству сообщений или суммарному числу символов) самые старые сообщения удаляются, при этом системный промпт сохраняется.
* **Потоковый вывод (Streaming):** Ответы от модели выводятся в консоль в реальном времени, по мере генерации токенов.
* **Интеграция файлов в чат:** Вы можете прикрепить содержимое любого локального текстового файла к вашему сообщению, используя синтаксис `@::путь/к/файлу::` (поддерживаются файлы до 5 МБ).
* **Интерактивный режим обработки чанками:** Команда `/file_chunk` позволяет обрабатывать большие тексты по частям (по абзацам или количеству символов), не переполняя контекстное окно модели.
* **Безопасное прерывание:** Долгая генерация ответа модели может быть прервана пользователем комбинацией `Ctrl+C` без полного падения программы.

## Структура проекта
* `src/main.py` — точка входа, обработка ввода/вывода (View/Controller).
* `src/ai_client.py` — клиент для взаимодействия с OpenAI API (Model).
* `src/context_mgr.py` — логика обрезки контекста.
* `src/file_mgr.py` — чтение файлов и генерация чанков.
* `src/commands.py` — обработка специальных команд бота.
* `src/config.py` — безопасная загрузка конфигурации.
* `tests/` — unit-тесты (pytest).

## Установка и запуск

1. **Клонируйте репозиторий и перейдите в папку проекта:**
   ```bash
   git clone https://github.com/SysoevaYulia/mipt_homeworks_2026.git
   cd project
   ```

2. **Создайте виртуальное окружение и установите зависимости:**
   ```bash
   python -m venv venv
   source venv/bin/activate  # Для Windows: venv\Scripts\activate
   pip install -r requirements.txt # в файле requirements.txt находятся необходимые библиотеки
   ```

3. **Настройте конфигурацию:**
   Приложение может брать настройки из переменных окружения или из файла `config.yaml` (переменные окружения имеют приоритет).
   
   *Пример файла `config.yaml` (создайте его в корне проекта):*
   ```yaml
   api_key: "your_api_key_here"
   api_host: "http://localhost:11434/v1/"
   limit_message: 10
   limit_chars: 4000
   temperature: 0.7
   model: "gpt-3.5-turbo"
   system_prompt: "You are a helpful Python developer assistant."
   ```

4. **Запустите приложение:**
   ```bash
   python src/main.py
   ```

## Команды бота
Прямо во время диалога с ассистентом вы можете использовать следующие команды:
* `\q` — завершить работу программы.
* `/reset` — очистить историю диалога и консоль, начать новую сессию.
* `/file_chunk` — активировать режим обработки файла по частям.
  * Флаги: `paragraph=N` (резать по абзацам), `len=N` (резать по символам), `-y` (автоматическое подтверждение для перехода к следующему чанку).
  * Пример: `/file_chunk paragraph=3 -y`

## Тестирование и линтеры

Код покрыт Unit-тестами с использованием `pytest` и `unittest.mock` (покрытие составляет 54%).

**Запуск тестов и генерация HTML-отчета о покрытии:**
```bash
python -m pytest tests/ --cov=src --cov-report=html
```
*Отчет будет доступен в файле `htmlcov/index.html`.*

**Проверка стиля кода (Ruff):**
```bash
ruff check src/
```

**Проверка статической типизации (Mypy):**
```bash
mypy src/
```
